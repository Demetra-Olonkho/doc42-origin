Согласен. Более того, после твоего уточнения я бы сформулировал это как отдельный принцип документации проекта.

Причина не только в удобстве чтения.

Если проект изначально развивается на русском языке, то именно русские формы терминов являются первичными носителями смысла. Английские формы в таком случае выступают не заменой, а официальными эквивалентами.

То есть в русскоязычном документе правильно писать:

```text
Деметра Олонхо (Demetra Olonkho)

Издательский конвейер (Publishing Pipeline)

Виртуальная настольная игровая система (Virtual Tabletop, VTT)

Модель знаний Олонхо (Olonkho Knowledge Model)

Уровень обмена Ирида (Irida Exchange Layer)

Механизм согласования Ирина (Irina Reconciliation Engine)
```

а не наоборот.

При этом в англоязычном документе уже достаточно:

```text
Demetra Olonkho
Publishing Pipeline
Virtual Tabletop (VTT)
Olonkho Knowledge Model
Irida Exchange Layer
Irina Reconciliation Engine
```

без обязательного дублирования русских терминов.

---

Я бы даже добавил это в будущий DEM-0003 отдельным разделом:

```text
Правила употребления терминологии

В русскоязычных документах:

    Русский термин (English Term)

В англоязычных документах:

    English Term

если иное не требуется контекстом документа.
```

---

Соответственно, начало DEM-0000 должно выглядеть примерно так:

```text
# 2. Наименование проекта

Полное наименование проекта:

    Деметра Олонхо (Demetra Olonkho)

Краткое наименование проекта:

    Деметра (Demetra)
```

А дальше аналогично по всему документу:

вместо

```text
Demetra Olonkho представляет собой универсальную платформу управления знаниями.
```

следует писать

```text
Деметра Олонхо (Demetra Olonkho) представляет собой универсальную платформу управления знаниями.
```

при первом упоминании.

Затем уже можно использовать просто:

```text
Деметра Олонхо
```

без повторения английского варианта.

Вместо:

издательские конвейеры (Publishing Pipeline);
виртуальные настольные игровые системы (VTT);

поскольку аббревиатура VTT сама по себе не является названием технологии.

Таким образом, для DEM-0000 появляется дополнительное правило:

```text
При первом упоминании термина:

Русский термин (English Term)

При последующих упоминаниях:

Русский термин
```

и это правило затем будет формализовано в DEM-0003.

Я считаю это очень хорошим решением, потому что оно поддерживает заявленный статус русского как первого рабочего языка проекта и одновременно не создаёт проблем для подготовки английских версий документов. Более того, это поможет нам в дальнейшем при формировании официального глоссария проекта, который всё равно придётся создавать довольно рано.
