Здесь я бы предложил сразу решить один важный вопрос.

Что именно должен представлять собой «Паспорт темы»?

Я вижу два возможных варианта.

### Вариант А. Паспорт конкретного обсуждения

То есть документ, описывающий именно эту тему.

Например:

```text
Тема:
Организационное положение проекта Деметра Олонхо

Статус:
Открыта

Цель:
Определение организационного положения проекта

...
```

Плюсы:

- проще;

- сразу решает текущую задачу.

Минусы:

- потом придётся придумывать формат заново для других тем.

---

### Вариант Б. Паспорт темы как шаблон будущей системы

То есть уже сейчас оформить его так, как будто это первый экземпляр будущего стандарта.

Например:

```text
TOPIC-0001

Наименование:
...

Тип:
...

Источник:
...

Связанные проекты:
...

Связанные документы:
...

Принятые решения:
...

Открытые вопросы:
...

Статус:
...
```

Плюсы:

- сразу появляется стандарт;

- потом можно использовать для любых обсуждений;

- хорошо вписывается в будущую документационную систему Деметры.

Минусы:

- чуть объёмнее.
